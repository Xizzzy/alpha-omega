## Options Considered

### Use It As-Is

**Omega's view:**
The repo is already bootstrapped enough to self-develop: `.alpha-omega/` exists, `AGENTS.md` and `CLAUDE.md` are present, `context_builder` auto-loads project docs and AO memory, and both CLI wrappers passed unrelated non-interactive JSON smoke tests. The main missing piece is history, not structure.
- Pros: Matches the current minimalist stdlib-only design; No code changes needed; Operational prerequisites are present: `claude` and `codex` binaries exist and wrappers work on trivial prompts; Docs and implementation are aligned on project-local memory and self-development intent
- Cons: Self-development debates do not automatically see `ao.py` or `core/*.py` unless `--extra` is supplied; The decision log is empty, so the self-improvement loop has not been exercised yet; This is 'usable' bootstrap, not 'frictionless' bootstrap

### Add Self-Dev Preset

**Omega's view:**
Treat the tool as mostly bootstrapped but tighten the last gap: when debating the alpha-omega repo itself, automatically include `ao.py` and core modules, or add an explicit `--self-dev` preset. That makes self-development first-class instead of relying on the user to remember `--extra` for internal code.
- Pros: Directly fixes the biggest self-hosting weakness: code context is not automatic; Small change with high leverage; Preserves the existing protocol and memory model; Makes debates about `core/sigma.py`, `core/protocol.py`, and wrappers more reliable
- Cons: Adds a repo-specific behavior or a new flag that slightly complicates the CLI; Can increase prompt/context size if done too broadly; Needs a clear rule to avoid auto-including irrelevant files

### Require Bootstrap Debate First

**Omega's view:**
Do not call the tool properly bootstrapped until it has completed and saved at least one self-development debate and decision entry. This treats bootstrap as a proven closed loop rather than just correct files and commands.
- Pros: Creates concrete evidence that the whole loop works end-to-end; Seeds `.alpha-omega/decisions.md` immediately; Turns 'self-development' from aspiration into actual practice
- Cons: This is process, not architecture; A first saved debate can still be low quality if it lacks code context; It does not fix any underlying ergonomics
