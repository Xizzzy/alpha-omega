**Omega's view:**

- Pros: Packaging is basically functional: `pyproject.toml:5`-`36` installs an `ao` console script, and `python3 -m pip wheel . --no-deps` succeeded.; I did not find classic shell injection: subprocesses are invoked with argv lists, not `shell=True` (`alpha_omega/primitives.py:69`, `:164`, `alpha_omega/context_builder.py:216`, `alpha_omega/cli.py:225`, `:316`, `:722`).; Bare CLI entry points work: `python -m alpha_omega`, `python ao.py`, and `--help` all execute.

### hardening sweep

**Omega's view:**
Block PyPI until a short prepublish hardening pass fixes the concrete defects already visible in this repo: path containment, review coverage, lock cleanup, backward-compatible memory indexing, test discoverability, and packaging/docs drift. This is the smallest option that turns the tool from interesting prototype into defensible public alpha.
- Pros: Fix path traversal/exfiltration first: validate `session_id` against a strict pattern and enforce `realpath` containment for `extra_files` (`alpha_omega/cli.py:586`, `alpha_omega/context_builder.py:87`).; Fix lock and subprocess error handling: wrap implement lock lifecycle in `try/finally` (`alpha_omega/cli.py:621`-`689`), and stop treating nonzero subprocess exits as usable content (`alpha_omega/primitives.py:85`, `:164`-`206`).; Fix review correctness: include untracked files via `git ls-files --others --exclude-standard` or equivalent and surface non-git-repo as an explicit error (`alpha_omega/context_builder.py:142`-`156`, `:219`, `alpha_omega/cli.py:488`).; Fix memory/API mismatch: `cmd_recall` claims to search decisions/debates/reviews (`alpha_omega/cli.py:946`), but `build_index` only reads `sessions/` and `reviews/` (`alpha_omega/memory.py:77`-`130`). In this repo the index sees 6 debate docs while `.alpha-omega/decisions.md` contains 13 decisions.; Fix testing before publish: the only test file is `tests/test_cli_implement.py:1`, and default `python -m unittest` currently reports `Ran 0 tests`; only `discover -s tests` finds 3 tests. Add discoverability and cover review/memory/security regressions.; Fix packaging/docs drift: `README.md:20`, `:89` and `AGENTS.md:15` are stale relative to the current package; `pyproject.toml:10` declares MIT but the built wheel contains no license text file.

## Debate Exchange

### Alpha's critique of Omega

**Steelman:** Omega identified concrete security defects (path traversal in session_id and --extra) and reliability gaps (untracked files invisible to review, stale locks) that make the current codebase unsafe for public PyPI distribution, and correctly argues that a focused hardening pass is the minimum viable path to a defensible alpha release.

- [moderate] The 'hardening sweep' option conflates genuine blockers (path traversal, lock cleanup) with completeness issues (memory index gaps, test coverage, docs drift). The memory indexing mismatch doesn't prevent safe operation—it just means recall may miss some historical decisions. Similarly, test discoverability and README staleness are quality issues, not publish blockers. This scope creep risks delaying the release for improvements that don't materially change the safety profile.
- [minor] Omega didn't assess whether this tool has existing PyPI users or quantify backward-compatibility impact. If 0.2.0 would be the first public release, strict session_id validation breaks nothing. If there are existing users with ad-hoc session files, the breaking change deserves explicit communication strategy, not just a buried 'con' mention.
- [minor] The --extra path traversal finding needs threat model clarification. Is 'ao debate --extra /etc/passwd' a vulnerability or a power-user feature? If debates are always local-only (not uploaded/shared), and the user explicitly specified the path, then reading outside project scope might be intentional for including global configs or cross-project context. The session_id traversal is clearly a bug (user doesn't control that string in normal operation), but --extra deserves nuance.
- *Concession:* Omega correctly identified that my original memo failed to produce substantive analysis due to tool execution error, leaving them to do the full investigation work alone.
- *Concession:* The session_id path traversal (cli.py:586, :609, :682) is a genuine security bug I would have missed without their code review, and it should block any public release.
- *Concession:* The lock cleanup gap (cli.py:621-689 lacking finally) is a practical reliability blocker that would cause user frustration—stale locks after crashes would make 'ao implement' appear broken until manual intervention.

## Assumptions

- Backward compatibility with existing `.alpha-omega/decisions.md` history matters, so recall/contradictions should not silently ignore older sessions.

- This is a public PyPI publish for unknown users, not a private alpha for trusted collaborators.
